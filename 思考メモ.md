問題文を読んで「内部的にはposという値が使われており...」らへんがよく分からなかったがLinked List cycleと変わらずサイクルがあるか判定すればよく、ある場合には  
始まる場所を返せばよい  
一瞬逆回りとかがよぎったが連結リストなので一方向だからこの方針はない  
以下のような絵を考える  
〇-〇-〇  
     |   |  
     〇ー〇  
自然言語化  
1.)サイクルがあるかの判定にはfast/slow法が使えそう  
2.)fastとslowが出会うのはサイクルの中  サイクルの開始位置-head=a  
3.)衝突位置-サイクル入口=b  サイクルの残り=c  サイクルの長さL=b+c  
ここで衝突時の移動距離はそれぞれ  
slow = a+b  
fast = 2(a+b)  
fast - slow =a+b  
これは余分に余った分なので、a+b=kLより、a=(k-1)b+kc  
ただ、k=1の場合で考えてみればわかるがbの項は周回分なので、a≡c (mod L)  
4.)つまり、衝突点を見つけてそのあと片方を引き戻して同じ速度で進めればよさそう
```
class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        slow = head
        fast = head

        # サイクルがあるか判定（衝突点を見つける）
        while fast is not None and fast.next is not None:
            slow = slow.next
            fast = fast.next.next

            if slow is fast:
                # 1つをheadに戻す
                slow = head

                # 同じ速度で進める
                while slow is not fast:
                    slow = slow.next
                    fast = fast.next

                # 出会った場所がサイクル開始点
                return slow

        # サイクルなし
        return None
```
